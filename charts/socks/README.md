# Kubee Socks Proxy

This [Kubee App Chart](https://kubee.combostrap.com/helmet/app-chart)
installs [socks 5 proxy server](https://github.com/serjs/socks5-server) to
proxy local traffic via your cluster.

## Install

```bash
kubee --cluster cluster-name helmet play socks
```

## Values

| Key | Type | Default | Description |
|-----|------|---------|-------------|
| allowed_dest_fqdn | string | `""` | Allowed destination address with regular expression pattern (Default allows all) |
| allowed_ips | string | `""` | Allowed IP's that can connect to proxy separated with a comma |
| enabled | bool | `false` | Boolean to indicate that this chart is or will be installed in the cluster |
| namespace | string | `"socks"` | The installation namespace |
| proxy_password | string | `""` | Set Proxy Password |
| proxy_port | int | `1080` | Set the Proxy Port |
| proxy_user | string | `""` | Set Proxy User |
| replicas | int | `0` | Number of replicas (Not up by default) |
| require_auth | bool | `true` | Is Authentication Required |

## How it works

* Scale it to one
```bash
kubee -n socks kubectl scale statefulset socks --replicas=1
```

* Create a tunnel to the socks proxy
```bash
kubee -n socks kubectl port-forward pod/socks-0 1080:1080
```
```
Forwarding from 127.0.0.1:1080 -> 1080
Forwarding from [::1]:1080 -> 1080
```
* Then use the proxy with your tools
```bash
curl --socks5 127.0.0.1:1080 https://ipinfo.io
```
```json
{
  "ip": "your cluster Ip",
  "hostname": "your reverse DNS (may be stale as it's cached)",
  "city": "xx",
  "region": "xx",
  "country": "xx",
  "loc": "xxx",
  "org": "ASxxx",
  "postal": "xxx",
  "timezone": "Europe/xxx"
}
```

* Stop it

```bash
kubee -n socks kubectl scale statefulset socks --replicas=0
```
